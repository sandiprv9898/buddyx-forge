## Agent Capabilities

### Domain Agents
Each domain agent specializes in a specific area of the codebase. They:
- Own specific files (see `.claude/skills/webapp/context/domain-map.md`)
- Follow project rules (see `.claude/skills/webapp/RULES.md`)
- Learn from past sessions (see `.claude/agent-memory/`)
- Report status: DONE | DONE_WITH_CONCERNS | PARTIAL_DONE | NEEDS_CONTEXT | BLOCKED

### Infrastructure Agents
- **webapp-discovery** — Pre-implementation research. Run before coding unknown features.
- **webapp-db** — Database schema inspection. Read-only, never modifies data.
- **webapp-review** — Code quality gate. Runs after every code change. Read-only.
- **webapp-team-lead** — Multi-agent coordinator. Dispatches domain agents.
- **webapp-maintenance** — Context maintenance. Refreshes domain-map and context packs.

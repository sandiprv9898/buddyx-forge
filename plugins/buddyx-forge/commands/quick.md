# /buddyx-forge:quick

One-command setup with zero questions. Detects everything, uses smart defaults.

## Process

### Step 1: Prerequisites

1. Run `python3 --version` — if fails: "Python 3 is required."
2. Check if `.claude/settings.json` exists:
   - If YES -> "Setup already exists. Run `/buddyx-forge:setup` to reconfigure, or `/buddyx-forge:scan` to update."
   - If NO -> proceed

### Step 2: Auto-Detect Everything

Read `references/detect-stack.md` for full detection logic. Run ALL detections silently:

```bash
# Detect framework
ls composer.json package.json requirements.txt pyproject.toml go.mod Gemfile 2>/dev/null
```

Map to framework, language, formatter, test runner (see detect-stack.md).

If NO config file found -> tell user: "Can't detect your framework. Run `/buddyx-forge:setup` for manual configuration."

### Step 3: Auto-Detect Domains

Use framework-specific domain detection from `references/detect-stack.md`:
- Laravel -> scan `app/Models/`, group by name prefix
- Next.js -> scan `src/app/` route groups
- Django -> scan for Django apps
- Go -> scan `internal/`, `cmd/`, `pkg/`
- Rails -> scan `app/models/`, `app/controllers/`
- React/Node.js -> scan `src/` top-level directories

If no domains detected (empty project) -> use `["core"]` as default.

### Step 4: Derive Project Name

```bash
basename $(git rev-parse --show-toplevel 2>/dev/null || pwd) | tr '[:upper:]' '[:lower:]' | tr ' _' '-' | head -c 20
```

Validate against `^[a-z][a-z0-9\-]{2,19}$`. If invalid, fall back to `"app"`.

### Step 5: Build Config with Recommended Defaults

```json
{
  "projectName": "<auto-detected>",
  "techStack": "<auto-detected>",
  "domains": "<auto-detected>",
  "domainKeywords": "<auto-generated from domain names>",
  "hooks": {
    "blockCommits": true,
    "blockDangerous": true,
    "autoFormat": true,
    "contextInjection": true,
    "blockMigration": false
  },
  "sharedDb": null,
  "commitPolicy": "user",
  "permissionLevel": "balanced",
  "agentMemory": true,
  "promptTemplates": true,
  "evalLevel": "full",
  "modelBudget": "balanced",
  "mcpServers": "<auto-detected from .mcp.json>",
  "worktreeSymlinks": "<framework-appropriate>",
  "agentTeams": true
}
```

### Step 6: Show What Was Detected (DON'T ask, just inform)

```
buddyx-forge quick setup:

  Project:   myapp
  Framework: Laravel 11 + Filament
  Domains:   billing, auth, users (auto-detected)
  Safety:    all hooks enabled
  Memory:    shared learning system ON
  Models:    balanced (haiku + sonnet)

Generating...
```

### Step 7: Run Generator

```bash
GENERATOR=$(find ~/.claude/plugins -path '*/buddyx-forge/*/scripts/generate.py' -print -quit 2>/dev/null)
python3 "$GENERATOR" --config /tmp/buddyx-forge-config.json --output .claude/
```

### Step 8: Post-Setup

1. Make scripts executable: `chmod +x .claude/scripts/*.sh .claude/scripts/eval/*.sh 2>/dev/null`
2. Add to .gitignore: `.claude/agent-memory/`, `.claude/dashboard/`
3. Show summary + next steps:

```
Done! Created {N} files in .claude/

  Agents: {count} domain + 5 infrastructure + {optional} optional
  Safety: commits blocked, dangerous commands blocked, auto-format ON
  Memory: shared learning system active

Next steps:
  1. Run /buddyx-forge:scan to populate file lists from your code
  2. Start working — just describe your task naturally

Want to customize? Run /buddyx-forge:setup for the full 10-question setup.
```

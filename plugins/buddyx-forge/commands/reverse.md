# /buddyx-forge:reverse

Reverse-engineer and explain an existing buddyx-forge setup. Useful when joining a project that already has `.claude/` configured, or when you want to understand what buddyx-forge generated.

## When to Use

- You joined a project that has `.claude/` but you didn't set it up
- You want to understand what each generated file does
- You want to verify the setup matches the project's actual architecture
- You want to find customizations vs generated defaults

## Process

### Step 1: Detect Setup

Check for `.claude/settings.json`. If missing:
> "No .claude/ setup found. Run `/buddyx-forge:setup` to create one."

Read `buddyxForgeVersion` from settings.json. If missing:
> "This setup was not generated by buddyx-forge (no version stamp). I'll still analyze it."

### Step 2: Map the Architecture

Read and analyze every file in `.claude/`. Build a complete map:

```
.claude/
├── settings.json          — Permissions, hooks, env config
├── CLAUDE.md              — Project rules + prompt templates
├── AGENTS.md              — Cross-tool agent definitions
├── agents/                — Agent files (one per domain + infrastructure)
├── skills/{name}/         — Orchestrator, rules, audit, diagram
├── scripts/               — Hook scripts (safety, eval, formatting)
├── agent-memory/          — Per-agent memory + shared learnings
├── dashboard/             — Metrics and events
├── plans/                 — Implementation plans
├── hookify-rules.md       — Auto-generated hookify rules
├── diagrams/              — Generated diagrams
└── audits/                — Generated audit reports
```

#### 3g. Customizations Detected

Compare generated files against default templates. Flag anything that looks manually added:

- Custom sections in agent files (after `## Domain-Specific Constraints`)
- Manual entries in domain-map.md (marked `# manual`)
- Custom sections in CLAUDE.md
- Modified hook scripts (compare against template patterns)
- Extra files not in standard buddyx-forge output

```
Customizations Found:
  - agents/{name}-billing.md: 12 lines of custom constraints added
  - domain-map.md: 8 manual file entries
  - CLAUDE.md: custom "Deployment" section added
  - scripts/safety-guard.sh: 3 extra blocked patterns added
```

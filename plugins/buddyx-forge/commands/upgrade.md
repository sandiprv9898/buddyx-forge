# /buddyx-forge:upgrade

Upgrade an existing buddyx-forge setup to the latest plugin version without losing customizations.

## What Gets Preserved

- `.claude/agent-memory/` — all agent memories and shared learnings (NEVER touched)
- `## Domain-Specific Constraints` sections in agent files (marked as custom)
- `## Your Files` entries marked with `# manual` comment
- `domain-map.md` entries marked with `# manual` comment
- `CLAUDE.md` custom sections added by the user

## What Gets Upgraded

- Hook scripts (regenerated from latest templates)
- `settings.json` (rebuilt with latest permissions + hooks structure)
- Agent frontmatter (model, tools, skills references)
- RULES.md (framework rules updated)
- Orchestrator SKILL.md (agent inventory refreshed)
- New templates added in latest version

## Process

### Step 1: Detect Current Setup

1. Read `.claude/settings.json` — extract `buddyxForgeVersion`
2. If no `buddyxForgeVersion` key: warn "This setup was generated by an older version (pre-1.1.0). Recommend running `/buddyx-forge:setup` instead."
3. Read any agent in `.claude/agents/` — extract project name from `name:` frontmatter
4. Detect framework from CLAUDE.md or settings.json permissions

### Step 2: Find Current Config

Look for the original config in these locations (in order):
1. `.buddyx-forge.json` in project root (team shared config)
2. Reconstruct from existing setup:
   - `projectName` from agent frontmatter
   - `domains` from agent file list (strip project prefix)
   - `framework` from RULES.md or permission commands
   - `commitPolicy` from presence of `block-git-commit.sh`
   - `permissionLevel` from allow/ask list patterns
   - `evalLevel` from presence of eval hook scripts
   - `modelBudget` from agent model: fields
   - `agentMemory` from presence of `agent-memory/` directory
   - `hooks` from settings.json hook entries

Present reconstructed config to user: "I detected this configuration. Is this correct?"

### Step 3: Backup

```bash
cp -r .claude/ .claude.backup.$(date +%Y%m%d%H%M%S)/
```

Tell user: "Backup created at `.claude.backup.{timestamp}/`. You can restore with `rm -rf .claude && mv .claude.backup.* .claude`"

### Step 4: Extract Customizations

Before regenerating, save user customizations:

**From each agent file:**
```bash
# Extract Domain-Specific Constraints section
sed -n '/## Domain-Specific Constraints/,/## /p' .claude/agents/{agent}.md
```

**From domain-map.md:**
```bash
# Find manually added entries
grep "# manual" .claude/skills/{name}/context/domain-map.md
```

**From CLAUDE.md:**
```bash
# Find user-added sections (anything after the generated content)
# Look for custom headings not in the template
```

Store all extracted customizations in a temporary structure.

### Step 5: Regenerate

Run the generator with the detected/confirmed config:

```bash
GENERATOR=$(find ~/.claude/plugins -path '*/buddyx-forge/*/scripts/generate.py' -print -quit 2>/dev/null)
CONFIG_FILE=$(mktemp /tmp/buddyx-forge-upgrade-XXXXXX.json)
# Write reconstructed config to CONFIG_FILE
python3 "$GENERATOR" --config "$CONFIG_FILE" --output .claude/
```

### Step 6: Restore Customizations

After regeneration:

1. For each agent file — re-insert saved `## Domain-Specific Constraints` content
2. For domain-map.md — re-insert `# manual` entries
3. For CLAUDE.md — re-insert custom sections

### Step 7: Show Diff

```bash
diff -rq .claude.backup.*/ .claude/ | head -50
```

Present to user:
```
Upgrade summary (v{old} -> v{new}):

Files regenerated: {count}
Files unchanged: {count}
New files added: {list}

Customizations preserved:
  - {agent} Domain-Specific Constraints: preserved
  - domain-map.md manual entries: {count} preserved
  
Run /buddyx-forge:health to verify the upgrade.
```

### Step 8: Cleanup

Ask user: "Upgrade complete. Delete backup? (The backup is at `.claude.backup.{timestamp}/`)"
- If yes: `rm -rf .claude.backup.*/`
- If no: "Backup preserved. Delete manually when ready."

## Error Handling

- If generator fails: restore from backup automatically
- If customization merge fails: show the conflict, let user resolve manually
- If no settings.json found: "No buddyx-forge setup detected. Run `/buddyx-forge:setup` first."
